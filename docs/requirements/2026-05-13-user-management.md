## 1. 需求概述

### 1.1 背景
为了完善后台管理系统的功能模块，需要新增用户管理功能，实现系统用户的全生命周期管理。

### 1.2 目标
新建一个用户管理菜单，提供用户列表展示、新增用户、编辑用户、删除用户、查询用户等核心功能，数据来源为 Mock 数据，为后续对接真实 API 做好准备。

### 1.3 适用范围
适用于 React + Ant Design 后台管理系统模板项目。

### 3.2 交互说明

#### 3.2.1 搜索区域
- **用户名输入框**: 支持模糊搜索，输入后点击查询按钮触发搜索
- **状态下拉框**: 选项包括"全部"、"启用"、"禁用"
- **查询按钮**: 点击后根据筛选条件刷新列表
- **重置按钮**: 清空搜索条件，恢复默认列表

#### 3.2.2 操作按钮区域
- **新增用户**: 点击打开新增用户弹窗
- **批量删除**: 选中表格数据后可点击，未选中时禁用状态

#### 3.2.3 列表表格
- **复选框**: 支持多选，用于批量操作
- **操作列**: 包含"编辑"、"删除"、"查看详情"按钮
- **分页**: 支持切换页码、每页条数

#### 3.2.4 新增/编辑弹窗
- 使用 Modal 组件，表单包含用户名、邮箱、手机号、状态、角色等字段
- 新增时密码为必填项，编辑时密码可选填

#### 3.2.5 删除操作
- 点击删除按钮弹出确认框
- 删除成功后刷新列表

## 4. 数据结构

### 4.1 用户数据模型

```typescript
interface User {
  id: string;              // 用户ID，唯一标识
  username: string;        // 用户名（唯一约束，创建后不可修改）
  password?: string;       // 密码（新增/编辑时使用，列表不返回）
  email: string;           // 邮箱
  phone: string;           // 手机号
  status: 'enabled' | 'disabled';  // 状态：启用/禁用
  role: string;            // 角色ID
  roleName?: string;       // 角色名称（展示用）
  createTime: string;      // 创建时间（格式：YYYY-MM-DD HH:mm:ss）
  updateTime: string;      // 更新时间（格式：YYYY-MM-DD HH:mm:ss）
  avatar?: string;         // 头像URL
  remark?: string;         // 备注
}
```

**业务约束**：
- `username` 字段必须唯一，不允许重复
- `username` 创建后不可修改
- 时间字段统一使用 `YYYY-MM-DD HH:mm:ss` 格式

### 4.2 角色数据模型

```typescript
interface Role {
  id: string;              // 角色ID
  name: string;            // 角色名称
  description?: string;    // 角色描述
}
```

> **说明**：角色数据为硬编码 Mock 数据，本次需求不包含角色管理模块开发。

### 4.3 列表查询参数

```typescript
interface UserQueryParams {
  username?: string;       // 用户名（模糊搜索）
  status?: 'enabled' | 'disabled' | '';  // 状态筛选
  page: number;            // 当前页码
  pageSize: number;        // 每页条数
}
```

### 4.4 列表响应数据

```typescript
interface UserListResponse {
  list: User[];            // 用户列表
  total: number;           // 总条数
  page: number;            // 当前页码
  pageSize: number;        // 每页条数
}
```

### 5.1 获取用户列表

**接口路径**: `GET /api/users`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 否 | 用户名，模糊搜索 |
| status | string | 否 | 状态：enabled/disabled |
| page | number | 否 | 当前页码，默认1 |
| pageSize | number | 否 | 每页条数，默认10 |

**响应数据**:
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "1",
        "username": "admin",
        "email": "admin@example.com",
        "phone": "13800138000",
        "status": "enabled",
        "role": "admin",
        "roleName": "管理员",
        "createTime": "2026-01-01 10:00:00",
        "updateTime": "2026-01-10 15:30:00"
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 10
  }
}
```

### 5.3 新增用户

**接口路径**: `POST /api/users`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名 |
| password | string | 是 | 密码 |
| email | string | 是 | 邮箱 |
| phone | string | 是 | 手机号 |
| status | string | 是 | 状态 |
| role | string | 是 | 角色ID |
| remark | string | 否 | 备注 |

**响应数据**:
```json
{
  "code": 200,
  "message": "创建成功",
  "data": {
    "id": "10"
  }
}
```

**错误响应**:
```json
{
  "code": 409,
  "message": "用户名已存在",
  "data": null
}
```

### 5.4 编辑用户

**接口路径**: `PUT /api/users/:id`

> **说明**：用户名（username）不可修改，请求参数中不包含 username 字段。

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| password | string | 否 | 密码，不传则不修改 |
| email | string | 是 | 邮箱 |
| phone | string | 是 | 手机号 |
| status | string | 是 | 状态 |
| role | string | 是 | 角色ID |
| remark | string | 否 | 备注 |

**响应数据**:
```json
{
  "code": 200,
  "message": "更新成功"
}
```

### 5.5 删除用户

**接口路径**: `DELETE /api/users/:id`

**删除权限规则**：
- 当前登录用户不可删除自己的账号
- 前端需在操作列判断当前用户ID，隐藏或禁用删除按钮

**响应数据**:
```json
{
  "code": 200,
  "message": "删除成功"
}
```

### 5.6 批量删除用户

**接口路径**: `POST /api/users/batch-delete`

**删除权限规则**：
- 当前登录用户不可删除自己的账号
- 若选中的ID中包含当前用户ID，需从删除列表中排除并提示

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| ids | string[] | 是 | 用户ID数组 |

**响应数据**:
```json
{
  "code": 200,
  "message": "批量删除成功"
}
```

### 5.7 获取角色列表

## 6. 交互流程

### 6.1 查看用户列表流程

```
用户进入页面 → 系统加载用户列表（默认第1页）→ 展示列表数据
     ↓
用户输入搜索条件 → 点击查询 → 系统重新加载数据 → 更新列表
     ↓
用户点击重置 → 清空搜索条件 → 加载默认列表
```

### 6.2 新增用户流程

```
用户点击"新增用户"按钮 → 弹出新增表单弹窗
     ↓
用户填写表单信息（用户名、邮箱、手机号、状态、角色等）
     ↓
用户点击"确定" → 系统校验表单
     ↓
校验通过 → 调用新增接口 → 接口成功 → 关闭弹窗 → 提示成功 → 刷新列表
     ↓
校验失败 → 显示错误提示 → 用户修正后重新提交
```

### 6.3 编辑用户流程

```
用户点击列表中的"编辑"按钮 → 弹出编辑表单弹窗 → 自动填充当前用户信息
     ↓
用户修改表单内容
     ↓
用户点击"确定" → 系统校验表单
     ↓
校验通过 → 调用编辑接口 → 接口成功 → 关闭弹窗 → 提示成功 → 刷新列表
     ↓
校验失败 → 显示错误提示 → 用户修正后重新提交
```

### 6.4 删除用户流程

```
用户点击列表中的"删除"按钮 → 弹出确认框
     ↓
用户点击"确定" → 调用删除接口 → 接口成功 → 提示成功 → 刷新列表
     ↓
用户点击"取消" → 关闭确认框
```

### 6.5 批量删除流程

```
用户勾选多个用户 → 点击"批量删除"按钮 → 弹出确认框
     ↓
用户点击"确定" → 调用批量删除接口 → 接口成功 → 提示成功 → 刷新列表
     ↓
用户点击"取消" → 关闭确认框
```
